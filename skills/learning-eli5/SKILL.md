# Explain Like I'm 5 (ELI5)

Guidelines, best practices, and a structured framework for explaining complex technical concepts in simple, accessible terms using everyday analogies and progressive complexity.

## Explanation Framework

### 1. Analogy Building
- Use everyday objects and experiences
- Relate to familiar concepts
- Create mental models
- Visual explanations

### 2. Progressive Complexity
- Start with simple overview
- Add layers of detail gradually
- Use examples at each level
- Build understanding step by step

### 3. Concrete Examples
- Real-world analogies
- Simple code snippets
- Step-by-step scenarios
- Before/after comparisons

### 4. Visual Descriptions
- Describe concepts visually
- Use diagrams in text form
- Spatial relationships
- Flow and process descriptions

### 5. Key Takeaways
- Core principles distilled
- One-paragraph summary
- Bullet point recaps
- Memory aids

## When to Apply

Use this framework when:
- Teaching beginners complex technical topics
- Simplifying difficult concepts for non-technical audiences
- Creating accessible documentation
- Breaking down intricate ideas

## Expected Output

When applying this framework, include:
1. Simple analogy-based explanation
2. Step-by-step breakdown
3. Concrete examples
4. Common misconceptions clarified
5. Related concepts to explore
6. Further reading recommendations
