# Explain Technical Concepts

Guidelines, best practices, and a structured framework for providing detailed technical explanations of concepts with examples, real-world applications, and advanced topics.

## Explanation Structure

### 1. Definition & Overview
- Clear definition
- Purpose and goals
- What problem it solves
- Historical context

### 2. Core Principles
- Fundamental concepts
- Key terminology
- Underlying theories
- Design philosophy

### 3. How It Works
- Step-by-step process
- Components and their roles
- Data flow and transformations
- State management

### 4. Implementation
- Code examples
- Common patterns
- Best practices
- Gotchas and pitfalls

### 5. Real-World Applications
- Use cases
- Industry examples
- Success stories
- Practical applications

### 6. Comparison
- Similar/different concepts
- Advantages over alternatives
- When to use vs not use
- Trade-offs

### 7. Advanced Topics
- Performance considerations
- Scaling aspects
- Security implications
- Future directions

## When to Apply

Use this framework when:
- Learning new technologies
- Explaining technical topics in depth
- Teaching complex concepts
- Documenting system architectures

## Expected Output

When applying this framework, include:
1. Comprehensive explanation
2. Multiple code examples
3. Visual diagrams (text-based)
4. Common use cases
5. Best practices guide
6. Resources for deeper learning
