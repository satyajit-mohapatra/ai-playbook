# Write Technical Blog Posts

Guidelines, best practices, and a structured framework for creating engaging, informative technical blog posts with proper structure, SEO considerations, and reader engagement.

## Blog Post Framework

### 1. Structure
- Attention-grabbing title
- Compelling introduction
- Problem/context setup
- Solution/explanation
- Code examples
- Best practices
- Conclusion and takeaways

### 2. Content Guidelines
- Clear, concise language
- Logical flow
- Code snippets with explanations
- Diagrams and visuals
- Practical examples
- Real-world applications

### 3. Technical Depth
- Start with basics
- Progress to advanced topics
- Explain the "why"
- Show alternatives
- Discuss trade-offs

### 4. Engagement Elements
- Questions to readers
- Interactive examples
- Challenges or exercises
- Further reading
- Author notes

### 5. SEO Considerations
- Keywords placement
- Meta description
- Header hierarchy
- Internal linking
- Alt text for images

## Blog Post Types

- Tutorial/How-to
- Deep dive analysis
- Problem solution
- Tool/library review
- Experience sharing
- Opinion piece

## When to Apply

Use this framework when:
- Creating tutorials or how-to guides
- Writing deep dive technical analyses
- Documenting problem solutions
- Reviewing tools, libraries, or technologies
- Sharing experiences or opinions on tech topics

## Expected Output

When applying this framework, include:
1. Complete blog post
2. Title options
3. Key takeaways
4. Code examples
5. Suggested images/diagrams
6. SEO recommendations
