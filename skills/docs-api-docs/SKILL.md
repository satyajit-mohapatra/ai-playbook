# Write API Documentation

Guidelines, best practices, and a structured framework for generating comprehensive API documentation including endpoints, authentication, request/response schemas, and code examples.

## Documentation Structure

### 1. Overview
- API purpose and description
- Base URL and versioning
- Authentication requirements
- Rate limiting information
- Error response format

### 2. Endpoints
For each endpoint include:
- HTTP method and path
- Description
- Request parameters (path, query, body)
- Response schema
- Status codes
- Example requests/responses

### 3. Authentication
- API keys
- OAuth flows
- JWT tokens
- Session-based auth
- Permission scopes

### 4. Models/Data Types
- Request/response schemas
- Field descriptions
- Data types
- Enum values
- Required vs optional fields

### 5. Code Examples
- cURL examples
- JavaScript/TypeScript
- Python
- Other popular languages

## When to Apply

Use this framework when:
- Documenting REST APIs
- Creating developer guides
- Onboarding API consumers
- Maintaining API reference documentation

## Expected Output

When applying this framework, include:
1. Complete API documentation
2. OpenAPI/Swagger specification
3. Code examples for all endpoints
4. Error reference guide
5. Authentication guide
6. Quick start examples
