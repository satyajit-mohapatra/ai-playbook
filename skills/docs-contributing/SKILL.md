# Write Contributing Guides

Guidelines, best practices, and a structured framework for creating comprehensive contributing guidelines for open source projects including setup, workflow, and community standards.

## Contributing Guide Structure

### 1. Welcome & Introduction
- Friendly greeting
- Why contribute to this project
- Types of contributions welcome
- Community values and code of conduct

### 2. Getting Started
- Prerequisites (tools, accounts, knowledge)
- Setting up development environment
- Cloning the repository
- Installing dependencies
- Running the project locally

### 3. Finding Issues
- How to find issues to work on
- Labels and their meanings
- Good first issues for newcomers
- Creating new issues
- Issue assignment process

### 4. Development Workflow
- Creating feature branches
- Coding standards and style guides
- Commit message conventions
- Pull request process
- Code review expectations

### 5. Testing
- Running tests
- Writing new tests
- Test coverage requirements
- CI/CD pipeline information

### 6. Documentation
- Updating documentation
- Docstring conventions
- README updates
- API documentation

### 7. Submitting Changes
- Pull request template
- PR description guidelines
- Linking related issues
- Addressing review feedback
- Merging process

### 8. Community
- Communication channels
- Contributor recognition
- Governance structure
- Support options

## When to Apply

Use this framework when:
- Creating contribution guides for open source projects
- Improving existing contribution processes
- Onboarding new contributors
- Setting up community standards

## Expected Output

Provide complete contributing guide with:
1. All sections above
2. Code examples
3. Links to related resources
4. Templates where applicable
5. Contact information
